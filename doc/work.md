# 大纲

依据doc/prompt1.md, doc/prompt2.md

## 原则

* 任何情况下，逻辑嵌套必须少于三层。
* 聚焦可能的性能瓶颈，向量化，静态化。以及减少内存分配。
* 所有同质功能的接口只保留一个性能最高、最易用的，使代码更清晰，不易误用。
* 禁止向后兼容性。
* 禁止使用const_cast（除非为了调用外部求解器的局部使用），mutable（除非为了缓存或者锁），friend，dynamic_cast等关键字。
* 删除冗余的成员变量、接口等。
* 比起创造新的类和接口，改造现有的类是更好的。
* 完成一块工作任务后：
  * 验证`doc/validation.md`全部四个案例。
  * 拒绝向后兼容性，强制改写所有调用处，让代码更简洁，对以后的扩展更通用。
  * 验证编译运行结果，移除所有向后兼容的或容易误用的的接口，防止冗余。
  * 提交一次代码，然后继续下一块工作。

## 工作任务1

* 目前代码的组织显得过于冗长和特化，存在大量的分支判断逻辑，硬编码逻辑，重复变量，让代码极其混乱。
* 你有必要对一些明显的问题进行彻底的重构。
* 消除不必要的转型动作和判断逻辑，尤其是在热循环内。
* 移除所有向后兼容的代码、冗余的接口、冗余抽象等。
